weights loaded from ai/weights.json once at game startup — see Training Pipeline
01
Spawn Timer Tick
The game's spawn timer fires on a regular interval, triggering the GM-AI pipeline. The interval itself is one of the parameters the GM can adjust.
2–6 second interval GM-controlled rate
02
gm_controller.gd — Collect Snapshot
Reads current game state and assembles the 20-feature vector. Features are in the exact same order the model was trained on. Rolling 10s stats (kills, hits, score) are tracked internally.
play_time score lives player position enemy counts total threat spawn budget kills / hits (10s)
03
Rolling Buffer — Last 8 Snapshots
Maintains a sliding window of the 8 most recent snapshots. Zero-padded on the left at game start. Gives the transformer temporal context — it can see trends, not just a single moment.
8 × 20 matrix zero-padded at start ~32 seconds of history
04
gm_inference.gd — Transformer Forward Pass
Runs the full transformer in pure GDScript — no Python process required at runtime. Z-score normalisation applied first, then the full encoder stack, then the classification head.
z-score normalise Input proj 20→64 Pos. Embedding 2× Encoder (MHA + FFN) 4 heads · d_head=16 FFN 64→128→64 Linear head → Softmax <1ms per inference
05
13 Action Weights
Softmax output is a probability distribution over all 13 GM actions. The highest-weight action is selected (argmax). The full distribution is available for the in-game visualizer overlay.
probability distribution sums to 1.0 argmax selection visualizer reads all 13
06
Apply GM Action
The chosen action immediately modifies spawning parameters. Takes effect on the next wave. The GM can apply pressure, grant relief, or send specialised enemy formations.
hold surge ease force_rest clear_screen budget +/- rate +/- force_elite force_chase force_swarm force_diagonal
game loop continues → back to step 01
