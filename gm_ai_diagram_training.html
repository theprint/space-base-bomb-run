Training Pipeline

Space Base Bomb Run — GM-AI

From raw gameplay to a deployable transformer in ~1 minute of training

Training Pipeline
01
Play the Game
Human plays normally. The built-in spawner runs unassisted while telemetry silently records in the background.
02
telemetry.gd — Record Telemetry
Every 4 seconds, captures a 20-feature snapshot of game state and saves a screenshot. Appends a row to a per-session CSV file automatically.
20 features / snapshot 4s interval PNG screenshots CSV per session
03
tools/labeler.py — Human Annotation
A Flask web UI shows each snapshot alongside its screenshot and key stats. The annotator selects 1–3 actions and adjusts their relative weights — producing a soft probability distribution, not a hard label.
human judgment soft labels 1–3 actions per row progress auto-saved
04
data/master_data.csv — Training Data
All labeled rows consolidated into one file. Each row contains 20 input features and 13 action-weight columns that sum to 1.0. Sessions are separated by play_time resets.
20 input features 13 action weights sums to 1.0 per row
05
gm_ai/train.py — Train Transformer
Trains a 2-layer transformer encoder against soft labels using KL divergence loss. Builds sliding windows of 8 consecutive snapshots. Adam + cosine LR schedule, dropout 0.3, early stopping at patience 25.
~55K parameters KL divergence loss seq_len = 8 d_model = 64 4 attention heads ~1 min on CPU
06
gm_ai/export_weights.py — Export
Converts the PyTorch checkpoint and z-score normalisation stats into a flat JSON structure readable by GDScript — no Python dependency at runtime.
PyTorch → JSON ~500 KB includes norm stats
07
ai/weights.json — Deployed Model
All transformer weights and normalisation stats packed into a single file. Committed into the Godot project and loaded once at game startup by gm_inference.gd.
weights + norm stats loaded at game startup
deployed → loaded by gm_inference.gd at game startup
