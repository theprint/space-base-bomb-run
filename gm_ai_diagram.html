Technical Architecture

Space Base Bomb Run — GM-AI System

How human-labeled gameplay data trains a transformer that controls difficulty in real time

Training Pipeline
01
Play the Game
Human plays normally. The built-in spawner runs unassisted while telemetry silently records.
02
telemetry.gd — Record Telemetry
Every 4 seconds, captures a 20-feature snapshot of game state and saves a screenshot. Appends a row to a session CSV.
20 features 4s interval PNG screenshots CSV per session
03
tools/labeler.py — Human Annotation
A Flask web UI shows each snapshot with its screenshot and stats. The human selects 1–3 actions and assigns relative weights — captured as a probability distribution.
human judgment soft labels 1–3 actions per row
04
master_data.csv — Training Data
All labeled rows consolidated. Each row: 20 input features + 13 action weight columns summing to 1.0. Sessions detected automatically by play_time resets.
05
gm_ai/train.py — Train Transformer
Trains against soft labels using KL divergence loss. Sliding windows of 8 snapshots. Adam + cosine LR schedule. Early stopping at patience 25.
~55K params KL divergence seq_len = 8 ~1 min on CPU
06
gm_ai/export_weights.py — Export
Converts the PyTorch checkpoint and normalisation stats into a flat JSON structure that can be parsed by GDScript without any Python dependency.
PyTorch → JSON ~500 KB
07
ai/weights.json — Deployed Model
All transformer weights + feature normalisation stats packed into a single file. Committed to the Godot project. Loaded once at game startup.
Inference / Gameplay
01
Spawn Timer Tick
The game's spawn timer fires on a regular interval, triggering the GM-AI pipeline. Interval is itself controlled by the GM.
2–6 second interval
02
gm_controller.gd — Collect Snapshot
Reads current game state and assembles the 20-feature vector. Identical feature order to what the model was trained on.
play_time score lives enemy counts threat spawn budget kills/hits (10s)
03
Rolling Buffer — Last 8 Snapshots
Maintains a sliding window of the 8 most recent snapshots. Zero-padded on the left at game start. Provides temporal context to the transformer.
8 × 20 matrix zero-padded
04
gm_inference.gd — Transformer Inference
Runs the full transformer forward pass in pure GDScript. No Python process required at runtime. Reads weights from ai/weights.json.
Input proj 20→64 Pos. Embedding 2× Encoder MHA (4 heads) FFN 64→128→64 Head + Softmax <1ms inference
05
13 Action Weights
Softmax output is a probability distribution over all 13 GM actions. The action with the highest weight is selected (argmax).
probability distribution sums to 1.0 argmax selection
06
Apply GM Action
The chosen action immediately modifies spawning parameters. Takes effect on the next wave.
hold surge ease force_rest clear_screen budget +/- rate +/- force_elite force_chase force_swarm
game loop continues → back to step 01
